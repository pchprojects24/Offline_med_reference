Coverage Review & Fill Plan

Current information gaps and a proposed fill order. No new clinical content has been added yet.

Current coverage snapshot

  • ACLS: 6 core adult algorithms plus 6 special-situation pages are already available.
  • Medications: 97 medication reference pages are available.
  • Procedures: 12 procedure pages are available.
  • Trauma / ATLS: an initial primary-survey reference page is now available.
  • Placeholder-only sections: Shipboard / At-Sea, Evacuation / Escalation, and Environmental / Operational Medicine still lack reference content.

Confirmed information gaps

  1. Trauma / ATLS: the initial primary survey is now covered, but secondary survey, trauma decision aids, and shipboard-specific trauma considerations are still missing.
  2. Evacuation / Escalation: there is still no offline guidance for MEDEVAC triggers, communication handoff structure, or escalation criteria.
  3. Shipboard / At-Sea: maritime-specific constraints, telemedicine workflows, and resource-limited care guidance are not yet present.
  4. Environmental / Operational: heat/cold injury, altitude, diving, and toxicology topics are still missing.
  5. Admin / Quick Reference: quick tables, contact workflows, and compact calculation aids are not yet available.
  6. Documentation mismatch: the repository README still describes some already-populated areas as future or placeholder content, so the project inventory is not fully up to date.
  7. Offline source depth: many medication and procedure pages end with online-only reference links, so deeper source material is not fully available without connectivity.

Proposed fill plan (for review before execution)

  1. Phase 1 — highest-value safety coverage: continue Trauma / ATLS expansion with secondary survey support and add Evacuation / Escalation checklists, because they close major offline decision-support gaps.
  2. Phase 2 — operational context: add Shipboard / At-Sea and Environmental / Operational quick references focused on scenarios likely to be encountered in remote settings.
  3. Phase 3 — rapid admin tools: add the Admin / Quick Reference section with contact workflows, compact tables, and small calculation aids that fit the current mobile-first layout.
  4. Phase 4 — cleanup and consistency: update README inventory, standardize section status labels, and flag where source material is still online-only.

Review guardrails

  • Keep all future additions offline-safe: relative links only, no external scripts, no dependency on network access.
  • Add content incrementally by section so each phase can be reviewed before broader expansion.
  • Preserve the existing concise checklist style used by the ACLS, medication, and procedure pages.

Planning artifact only — no gap-filling content from this plan has been implemented yet.
